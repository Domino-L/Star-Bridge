# StarBridge Roadmap

## 项目定位

星海舰桥是一个《星际公民》舰队协作应用。当前阶段的重点不是堆叠更多页面，而是让登录、舰队、小队、任务、Overlay、舰船数据库这些核心协作数据稳定可信，形成可以继续扩展的基础。

## 当前阶段：0.3.x 原型稳定期

目标是让小规模舰队可以完整测试：

- 用户登录后拥有明确身份。
- 用户可以创建、加入、离开舰队。
- 用户可以创建、加入、离开小队。
- 舰队指挥官和被授权成员可以管理舰队。
- 任务、计划、公告、成员状态、头像、队标、舰船数据库可以稳定同步。
- Overlay 可以可靠展示当前舰队/小队状态。
- 更新、反馈、邮件通知、服务器同步失败都有明确提示。

## P0 协作核心稳定包

这是下一阶段必须优先完成的稳定包。

### 1. 服务端权威数据边界

- 服务端是舰队、成员、小队、任务、计划、权限、日志的权威来源。
- 客户端只提交自己有权限修改的数据。
- 远端数据不能被旧的本地缓存覆盖。
- 清空服务器数据后，客户端必须清理失效的舰队、小队、头像、队标和用户状态缓存。

验收标准：

- 退出登录后不显示旧舰队和旧个人信息。
- 服务器清空后，客户端不会继续显示旧舰队。
- 新舰队不会继承旧舰队队标、头像、小队或成员缓存。

### 2. 成员与权限

- 舰队指挥官默认拥有全部权限。
- 自定义职称可以开放对应管理能力。
- 服务端必须校验权限，不能只靠客户端隐藏按钮。
- 撤销权限后，用户不能继续调用对应管理接口。

验收标准：

- 没权限的账号直接调用接口会返回 401/403。
- 有权限的账号可以执行对应管理操作。
- 权限变更会写入舰队日志。

### 3. 小队生命周期

- 每人只能创建一个小队。
- 小队指挥官离开时必须移交指挥权；若小队只有自己，则确认后解散。
- 空小队应被清理。
- 更换小队图标后，本地不能被旧远端快照覆盖。

验收标准：

- 小队图标更换后不会几秒内回退。
- 没有成员的小队不会长期残留。
- 小队指挥官离队后不会出现无指挥官小队。

### 4. 任务、计划、公告

- 当前任务、集结点、行动计划、公告全部进入服务器同步。
- “再次通知”应触发 Overlay 通知和邮件通知。
- 行动计划参与状态应同步并持久化。
- 已完成、取消、过期的任务和计划应进入历史记录。

验收标准：

- A 客户端发布任务，B 客户端 Overlay 和任务栏能收到。
- 计划参与按钮状态在重启后仍正确。
- 舰队日志记录任务发布、完成、删除、再次通知。

### 5. 审计日志

- 成员加入/离开、权限变更、任务发布、计划发布、小队创建/解散等事件写入舰队日志。
- 日志按时间从新到旧排序。
- 重复同步不能反复生成相同日志。

验收标准：

- 自动同步不会刷出重复“加入舰队”事件。
- 日志可筛选并能稳定显示最近事件。

## P1 前端体验整理包

- 首次启动引导。
- 登录/注册错误提示。
- 首页滚动、更新说明、反馈入口。
- 加入/退出舰队、小队的二次确认。
- 创建舰队和创建小队表单分步骤优化。
- 管理舰队界面按公告、成员、申请、任务、计划、舰船、日志、解散组织。

## P2 Overlay 表现包

- Overlay 预设：战斗、精简、指挥、自定义。
- 船厂风格：默认、铁砧、德雷克、南船座、武藏、十字军、圣盾、RSI、起源、奥波亚、埃斯佩里亚、盖塔克、未来等。
- 虚拟准星：开关、大小、粗细、颜色、透明度、风格联动。
- 自动按当前飞船厂商切换 Overlay 风格。

## P3 Log 解析与状态可信度

- 飞船状态：高/中/低可信度，不显示调试细节。
- 地点状态：地点、可能在、可能离开。
- 量子目标、到达、区域进入、Join PU 分区、玩家上下线作为组合证据。
- 用户可上传地点对照表。

## P4 舰船数据库

- 官网机库 HTML 导入。
- 单船和船包识别。
- 中文/英文舰船名映射。
- 按玩家、舰队、小队统计舰船。
- 后续考虑 WebView2 登录官网读取，但必须避免保存 RSI 账号密码。

## P5 发布、更新与运维

- 应用内更新弹窗显示版本说明、下载进度、安装提示。
- 更新完成后自动重启。
- 半自动服务器部署脚本使用 SSH key，减少手动输入密码。
- 服务端使用 HTTPS、Nginx 反代、环境变量文件和最小端口暴露。

- 服务端日志去重，避免同步重复事件。
- 小队快照合并增强，避免旧队标覆盖新队标。
- 客户端本地小队编辑保护，避免刚上传的图标/简介被远端旧快照回滚。
- 构建验证，并列出下一批稳定包任务。
